# Style Is Not Strategy

Many AI diagrams fail for a simple reason: they confuse visual style with expression strategy. A
diagram can have beautiful colors and still communicate poorly.

Style answers the question "what does this look like?" It includes color, type, line weight, spacing,
texture, and tone. Strategy answers "how should this idea be understood?" It includes the reading
path, grouping, contrast, hierarchy, and the diagram type.

If an agent only chooses a theme, it tends to produce the same board again and again: title, cards,
arrows, a few accent colors. The output looks different across themes, but the thinking is unchanged.

A better workflow separates the choices. First decide whether the task is a structured Mermaid
diagram or an explanatory SVG visual. Then choose the design style and layout style that support the
audience and mood.

For example, a leadership one-pager needs an assertion and three supporting pillars. A technical
flow needs Mermaid decisions and branch labels. A public article visual needs an SVG composition with
a memorable contrast or metaphor.

The lesson is practical: do not ask AI to "make it beautiful" first. Ask it to choose the right
communication shape. Beauty can then reinforce the message instead of hiding weak structure.
