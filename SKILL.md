# AgentDraw Skill

Use this skill when the user wants a local editable whiteboard, diagram, system map, journey map, process chart, architecture board, research synthesis wall, roadmap, or styled visual board that can be generated by an agent and then manually edited in a browser.

AgentDraw stores the board as structured `.agentdraw.json`, opens it in a local Excalidraw-based editor, validates common layout mistakes, and exports JSON/SVG/PNG.

## Install

```bash
git clone git@github.com:chenhg5/agentdraw.git
cd agentdraw
pnpm install
pnpm build
```

Open an example board:

```bash
pnpm agentdraw open examples/complex-agentdraw-workbench.agentdraw.json
```

Open without launching the system browser:

```bash
pnpm agentdraw open examples/complex-agentdraw-workbench.agentdraw.json --no-open
```

Validate a generated scene:

```bash
pnpm validate:scene path/to/board.agentdraw.json
```

## Agent Workflow

1. Understand the board purpose, audience, density, and desired tone.
2. Read [`CATALOG.md`](./CATALOG.md) and choose one style id.
3. Read only that style's `packages/styles/designs/<style-id>/design.md`.
4. Generate or patch a `.agentdraw.json` scene using editable primitives: text, rectangles, ellipses, diamonds, arrows, and lines.
5. Run `pnpm validate:scene <file>`.
6. Repair reported element ids until validation passes.
7. Open the board with `pnpm agentdraw open <file>` or `--no-open`.

## Scene Contract

Use this envelope:

```json
{
  "type": "agentdraw/scene",
  "version": 1,
  "title": "System map",
  "styleId": "system-formal",
  "providerId": "excalidraw",
  "elements": [],
  "appState": {},
  "files": {}
}
```

The scene is the editable output. The design system is guidance for creating it, not extra metadata that must be embedded in the file.

## Hard Rules

- Do not make screenshots when an editable board is expected.
- Do not use a style as a palette swap. Follow the selected `design.md`.
- Keep text editable and generously sized.
- Run validation before opening or delivering the scene.
- If the board intentionally uses hard shadows or decorative shapes, mark those elements with `customData.role: "shadow"` or `customData.role: "decoration"`.
